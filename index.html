Accessibility

ARIA Attributes

  • All inputs have appropriate aria-label or aria-labelledby
  • Error messages use aria-live="assertive"
  • Table headers have scope="col" or scope="row"
  • Sortable columns indicate direction with aria-sort

Keyboard Navigation

  • Tab navigation follows logical order
  • Input fields can be focused with keyboard
  • Table rows selectable with spacebar
  • Sortable columns activated with Enter key
  • Clear buttons activated with Enter/Space

Best Practices

Do's

  • Use clear and descriptive labels for inputs
  • Provide helpful validation messages
  • Ensure sufficient color contrast
  • Implement responsive design
  • Provide keyboard alternatives

Don'ts

  • Don't rely solely on color to convey information
  • Avoid placeholder text as label replacement
  • Don't make tables too complex
  • Avoid small click targets
  • Don't disable hover states on touch devices
